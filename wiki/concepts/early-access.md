# Early Access

A pre-go-live window during which participants can access the Transamerica platform before the full plan is live. Early access is optional and COM-coordinated, with the [[lm-dc]] providing the file and [[tc]] activating the feature.

## Definition

Some plans offer participants a period of early access — typically for enrollment purposes — before the conversion's official go-live date. This requires a specific early access file to be loaded into the system by the DC, and a system activation by TC.

## Workflow

1. COM requests the early access file template from [[lm-dc]]
2. DC provides the template
3. COM sends the template to the client for completion
4. Client completes and returns the file to COM
5. COM submits the completed file to DC
6. COM verifies with DC that the file has been submitted and loaded into the system
7. COM emails [[tc]] to turn on early access — this happens the day before early access starts
8. COM sends email to the plan notifying participants that early access is open

## Why It Matters

The early access window gives participants time to log in, review their accounts, and make enrollment elections before the conversion is fully complete. The timing is tight: TC must be notified the day before, and the client file must be in the system before that notification is sent. Any delay in the client returning the file compresses the window significantly.

## Evidence / Examples

- Early access is marked "if applicable" in the COM checklist — not every plan uses it.
- COM owns the full coordination chain; DC owns file loading; TC owns system activation.
