# Onboarding Package

An all-in-one Excel document used for certain plan types (especially PEP plans) in place of or alongside the PRD. Contains plan provisions, TOA, source details, and other conversion specs in a single file.

## Overview

The onboarding package consolidates into one Excel file what might otherwise be spread across multiple documents. For PEP (Pooled Employer Plans) and some other plan types, the onboarding package is the primary plan specification document.

## Contents

An onboarding package typically contains:
- **TOA (Transfer of Assets)** — the fund mapping table
- **Plan provisions** — eligibility, vesting, sources, investment options
- **Source details** — source names, codes, and parameters
- Other plan-specific fields

> Everything the [[prd]] covers, plus the [[toa]], in one Excel file.

## Plan Types That Use It

- **PEP — Pooled Employer Plans** — always use onboarding packages
- **All other plan types** — use the [[prd]] instead

PEP plans use the onboarding package. All other plan types use the PRD. The two are mutually exclusive — a plan uses one or the other, not both.

## PRD vs. Onboarding Package

| | PRD | Onboarding Package |
|--|-----|--------------------|
| Format | Typically a Word/PDF document | Excel workbook |
| TOA included | No (separate) | Yes (included) |
| Used for | Most plan types | PEP plans + others |
| Signed by client | Yes | Yes |
| Purpose | Same — detail plan provisions |

## DC Use

Same as with the PRD: DC focuses on eligibility, vesting, sources, and special strategies. The rest is for other departments.

The TOA being embedded means the DC can begin [[fund-mapping]] as soon as the onboarding package is signed (same rule as standalone TOA).

## Authorship and Template

- **This is a TA document** — not provided by the prior record keeper
- **COMs** put it together
- **Standard TA template** exists — COMs work from it
- **No additional DC-specific considerations for PEP plans** — DC processes them the same way as PRD-based plans
