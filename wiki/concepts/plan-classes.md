# Plan Classes

Groupings within a plan (e.g., employee categories) that determine contribution rules, eligibility, and other plan parameters. Defined in P3.

## Overview

Plan classes divide participants into groups within the plan — for example, hourly vs. salaried employees, or different divisions. Each class can have different eligibility rules, vesting schedules, or contribution sources. Classes are set up in [[p3]] and are informed by the [[prd]] or [[onboarding-package]].

## Two Types of Classes in P3

### Traditional / Regular Classes
Examples: union, non-alien resident, part-time
- Standard class types used across many plans
- **DC can update** existing names and codes
- **DC cannot add new** traditional classes — only update existing ones

### Special Classes
Examples: teachers, doctors, electricians — company/plan-specific groupings
- Non-standard classes defined by the plan sponsor
- **DC cannot add new special classes** — Doc Ops only
- DC can update existing special class names and codes
- If a new special class is needed: **email Doc Ops** with the request

## DC's Role

| Action | DC Can Do? |
|--------|-----------|
| Update existing class names | Yes |
| Update existing class codes | Yes |
| Add new traditional classes | No — update only |
| Add new special classes | No — Doc Ops only |

## P3 Navigation

> Plan → dropdown → Define Classes

## Interaction with Eligibility

- Some classes are **excluded from eligibility** entirely
- Other classes have **different entry rules** (different entry dates, service requirements, etc.)
- These rules come from the PRD/OB and are set up in P3 by Doc Ops — DC verifies them

## When Classes Come Up

Classes are identified during the [[prd]] / [[onboarding-package]] review. By the time P3 is set up, the classes should be defined. The DC verifies P3 has the correct classes and updates names/codes if needed.
