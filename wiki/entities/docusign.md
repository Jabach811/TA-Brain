# DocuSign

An electronic signature platform used by the [[com]] to send plan documents and onboarding agreements to clients for signature.

## Overview

DocuSign is an external e-signature service. In the Transamerica conversion workflow, the COM uses it as the primary channel for getting client signatures on formal plan documents. Multiple distinct document sets are routed through DocuSign depending on plan type.

## Key Facts

- External vendor platform (not a TA-built system)
- Used exclusively by COM in the documented workflow
- Two distinct use cases in PEP conversions:
  1. **OBP package** — On-Boarding Package, PEA, and Info Sharing Form
  2. **Plan documents** — AAA, Secure 2.0 amendment, and SPD

## Role in Plan Conversions

| Document Set | Plan Type | When |
|---|---|---|
| OBP + PEA + Info Sharing Form | PEP only | After OBP is created and tabs updated |
| AAA + Secure 2.0 + SPD | PEP only | After internal and client plan doc review |

Non-PEP document signature workflows are not yet documented — unclear whether DocuSign is used for non-PEP agreements.
