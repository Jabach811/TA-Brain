# P2

A back-end system region used primarily for creating and modifying transfer reference numbers. Separate from [[p3]] but connected — changes in P2 reflect in P3.

## Overview

P2 and P3 are two distinct system regions. P2 operates on the back end. DCs interact with P2 mainly during liquidation-related processing when ref numbers need to be created or modified.

## Key Functions

- **Creating transfer reference numbers** — one per fund on liquidation day (mapping/TIK)
- **Modifying ref numbers** — adjustments as needed

## Relationship to P3

P2 and P3 are connected:
- A change made in P2 **reflects in P3**
- P3 is the main working environment (plan details, participant data, processing)
- P2 is the back-end environment for ref number operations

Think of P2 as the engine room and P3 as the dashboard.

## When DC Uses P2

- **Liquidation day:** Create trans ref numbers (one per fund) before running the Day of Wire Informatica workflow
- **Advanced Employer:** Setting up the advanced employer account ref number (cash conversion)
- Any time a ref number needs to be created or adjusted

## Access and Permissions

- **Access method:** Separate URL interface (not within P3)
- **Who can access:** All DCs — no special restrictions

## Trans Ref Number Format

`YYYYMMDD` + ~7–8 random alphanumeric characters

Example structure: `20260414A3X7KL2`

## Other DC Operations in P2

Primarily ref number creation and modification. Possibly other operations — not fully documented yet.
