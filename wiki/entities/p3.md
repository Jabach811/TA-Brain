# P3

The main plan administration system. Central hub for plan configuration, participant data, processing, and queries.

## Overview

P3 is TransAmerica's primary recordkeeping platform. Almost every action in a plan conversion runs through P3 — from initial plan setup through balance posting, eligibility enabling, and final processing. DCs use P3 constantly throughout the onboarding lifecycle.

## Key Functions

### Plan Configuration
- Eligibility rules
- Vesting rules and schedules
- Source names and codes
- Plan classes (traditional and special)
- Divisions
- Investment options / fund lineup

### Classes in P3
Two types of classes exist:
- **Traditional/regular classes** — DC can update names and codes
- **Special classes** — DC cannot add; only [[internal-teams|Doc Ops]] can add new ones. DC can update existing ones.

See [[plan-classes]] for more detail.

### Processing
- Creating transfer reference (trans ref) numbers — done on P2 then reflected in P3, or directly on P3 (clarify)
- Processing participant balances (immediate / batch / online)
- Processing eligibility (enable by source)
- Reversing dummy participant details via the ROC tool

### EDS Integration
- P3 and EDS are connected: "P3 to EDS" setup is where you configure file layouts for the plan
- Must set up EDS layouts in P3 before testing files through EDS
- See [[eds]]

### Processing Modes (Critical)
| Mode | When Used | What It Does |
|------|-----------|-------------|
| Process Immediate + Batch | Cash conversion balance posting; mapping day-of-wire | Sends trades out |
| Online + No Hold | Mapping/TIK participant balance posting (after day-of-wire) | Books to ledger only, no trades |

> **Rule:** If trades have already gone out (day-of-wire workflow), use **Online, No Hold** when posting participant balances. Do NOT click Process Immediate.

### Eligibility
- Enable per source — only sources that will have contributions
- Must notify QA, TC, and COM before enabling
- See [[eligibility-and-deferrals]]

### ROC Tool
- Used to reverse dummy participant details after balance posting
- Part of P3

## Setup Timing

P3 setup (by Doc Ops) can happen before or after DC assignment. Earlier is better — DC needs P3 for eligibility rules, vesting, sources, classes, divisions. If P3 is set up, DC can start filling out specs immediately.

> P3 setup timing varies by team caseload. Don't assume it's ready.

## ROC Tool — Detail

The ROC tool is accessed from the **P3 menu bar**. It is the only place DCs are permitted to make direct changes to database tables (AQT is read-only — view only).

For dummy participant reversal after a mapping conversion:
- Query to run: **"ROC No Reversal"**
- Region: **case number starts with "T" → TDA; otherwise → CORP**
- Enter: **case number** + **SSN** (999-00-0000 for the dummy participant)
- The tool executes the reversal
- Afterwards: run the verification query to confirm the dummy's records are cleared

## Eligibility Navigation Path

> Plan → top left dropdown → Outsourcing Summary → click Eligibility → Enable screen
